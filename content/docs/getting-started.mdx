## Prerequisites

- [bun](https://bun.sh) (runtime and package manager)
- [just](https://github.com/casey/just) (optional, for workflow commands)
- An existing Claude Code installation (shan reads its artifacts)
- Optional: Codex if you want a mirrored skill view at `~/.codex/skills/` and `.codex/skills/`

## Install from source

```sh
git clone git@github.com:jasonkuhrt/shan.git
cd shan
just install
```

This runs `bun install` to fetch dependencies.

## Register globally

To make `bun x @jasonkuhrt/shan` available from any directory:

```sh
just install-global
```

This runs `bun link`, which registers the package globally.

## Install bundled skills

Shan ships five Claude Code skills that let you invoke shan commands from inside a Claude Code session (e.g. `/shan transcript print`). Install them at user scope:

```sh
just install-skills-user
```

This copies skill definitions into `~/.claude/skills-library/` and creates outfit symlinks in `~/.claude/skills/`. If `skills.agents` resolves to include `codex`, shan also reconciles the Codex mirror directories.

## Configure agents

Shan keeps Claude as the canonical outfit and can mirror that active set to Codex as whole-dir symlinks.

The default config is:

```json
{
  "version": 1,
  "skills": {
    "agents": "auto"
  }
}
```

Put that in `~/.config/shan/config.json`.

`"auto"` probes `claude` and `codex` on `PATH`, then caches the detected set in `~/.local/shan/cache.json` for 24 hours.

To pin the enabled set explicitly:

```json
{
  "version": 1,
  "skills": {
    "agents": ["claude", "codex"]
  }
}
```

After the next shan skill mutation, if the resolved agent set includes `codex`, these mirror directories will be reconciled to symlink to the canonical Claude outfit:

- `~/.codex/skills/`
- `.codex/skills/`

Shan owns the Codex mirror directories completely. If it finds a real Codex skills directory instead of the expected symlink, it migrates non-conflicting entries into the canonical Claude outfit and then replaces the Codex directory with the symlink. Same-name conflicts stop and surface an error.

## Import from skills.sh

Import skills from any `skills.sh` source into shan's own library/outfit model:

```sh
bun run shan -- skills install vercel-labs/agent-skills --skill typed-api-dx-review
bun run shan -- skills install vercel-labs/agent-skills --all --scope user
```

The importer is intentionally non-interactive: pass one or more `--skill` flags or `--all`. Imported skills are converted into shan-managed pluggable skills immediately, so `undo`, `move`, `doctor`, and agent mirrors keep working normally.

## First use: dump a transcript

Pick a recent Claude Code session and dump it as Markdown:

```sh
bun run shan -- transcript dump
```

Without a target argument, shan opens an interactive picker showing sessions for the current project. Select one. Shan writes a `.claude/transcripts/<session-id>.transcript.md` file with columnar headings you can navigate in your editor's outline view.

## First use: list skills

See which skills are active:

```sh
bun run shan -- skills list
```

The output shows core skills (always active), pluggable-on skills (symlinked from library), pluggable-off skills (available but not equipped), and the total character budget usage.

## What just happened

- `transcript dump` parsed the JSONL session file using Effect Schema, extracted entry types and summaries, and wrote formatted Markdown
- `skills list` read `~/.claude/shan/state.json` and the outfit/library directories to compute the effective skill set
- Resolved non-Claude agents are symlinked to the canonical Claude outfit rather than managed separately
- Both commands resolved file paths from Claude Code's standard locations (`~/.claude/projects/`, `~/.claude/skills/`)

## Running from the repo

If you skipped global registration, prefix commands with `bun run shan --`:

```sh
bun run shan -- transcript analyze dc8ffe42
bun run shan -- skills on playwright,linear
bun run shan -- task open
```
